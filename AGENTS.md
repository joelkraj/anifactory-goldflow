# AniFactory Goldflow

This repo is the clean longform production lane. It intentionally excludes legacy render paths, old visual heuristics, source-seed annotations, regex SFX production fallback, and multi-route compatibility behavior from the original AniFactory repo.

## Non-Negotiable Flow

Production moves through one artifact chain. Before step 0, run identity preflight is mandatory: decide the channel, stable series slug, stable run/week slug, exact episode id, title, image provider, audio target, and proof-vs-production intent, then write `run_identity.json` with `goldflow run preflight`.

0. Generate or obtain a polished narration story using the source-script workflow in `docs/workflows/source_script_generation_workflow.md`.
1. Ingest a polished narration story into `script_clean.md`.
2. Optional script polish/enhancement only when the operator explicitly asks for it, before approval, with the resulting script treated as a new candidate for review.
3. Manual review and operator approval for the exact final script hash.
4. Script pace check for the approved script. New production scripts target 210-220 spoken WPM; `script_pace_report.json` must exist for the exact approved hash before speakability or semantic planning.
5. Targeted production-readiness and speakability QA from the approved script. These passes may suggest TTS-only fixes for known problem areas, pronunciation mappings, and risk notes, but must not silently rewrite the story. Broad speakability is optional and must be explicitly requested.
6. Semantic scene annotation from the locked script and bibles.
7. Narrator-only voice plan by default. Voice planning requires current `script_speakability_report.json` and `tts_spoken_overrides.json` unless explicitly run in diagnostic bypass mode.
8. ModelsLab Qwen TTS and stitch.
9. Local Whisper word timing on the final stitched narration.
10. Audio pace check from Whisper timing. `narration_pace_report_<episode>.json` must pass at 210-220 actual WPM before timing bind or downstream production.
11. Review TTS and timing artifacts before downstream production.
12. Default narrator-only longform audio bed from the stitched narration/Qwen report. SFX, score, ambience, and transition SFX are opt-in variants, not default requirements.
13. Visual beat planning: build transcript-first editorial image beats from the final script, local Whisper word timing, and timed semantic scene context before prompt authoring. Semantic scenes are broad story truth; visual beats are local cut truth. Every beat must carry exact local narration excerpt text, local location, visible characters, mentioned-only characters, props/UI, editorial cues, a visual job, a suggested shot job, and beat-level advisory reference needs with generation-mode hints such as `standalone_ref`, `no_ref_needed`, `derive_from_first_clean_cut`, `derive_from_first_clean_wide_cut`, or `derive_from_best_cut`. These beat hints are not official reference locks; the visual reference LLM chooses final targets.
14. Visual reference planning: create `character_state_refs.json` and any style/location/action reference specs from locked script, bibles, and semantic scenes.
15. Manual agent/operator review and optimization of reference prompts. Character state refs become the definitive visual identity/wardrobe/state contract.
16. Reference generation in dependency order: style ref first only when needed, then character/location/action anchors.
17. Approve reviewed/generated references with `goldflow visual approve-refs` before visual prompt planning. Do not hand-edit `character_state_refs.json` approval state.
18. Visual prompt planning consumes approved references, current-scene facts, and visual beats.
19. Visual prompt review/fix pass checks the authored prompts against scene facts and approved refs. Use `visual review --auto-resolve true --max-resolve-iterations 2` by default so prompt-prose blockers route back to the LLM.
20. Visual prompt harden validates LLM-authored prompt/ref manifests, resolves approved ref paths, strips forbidden or unknown refs, enforces the four-reference cap, normalizes manifest/reference structure plus format-only prompt syntax, and writes a pre-imagegen sample sheet. It does not rewrite depicted prompt content.
21. Visual transition planning with silent transition SFX by default.
22. Image generation.
23. Render with intentional beat-selected transitions, final-script subtitles timed to Whisper, and one continuous mixed audio track.
24. Final QA: spot-listen narration cuts, spot-check prompt/image/ref consistency, verify transition style, verify subtitles are final-script text timed to Whisper, then `ffprobe` the final upload-safe render.
25. Package for upload: title, thumbnail, and description hooks should be generated after the story/render is understood, using proven channel packaging recipes and current episode truth.

Current migrated scope is source ingest, script approval, targeted speakability, semantic scene planning, the audio spine, Whisper timing, timing binding, SFX/score enrichment, longform audio mix, visual beat planning, visual reference planning, current-scene-only visual prompt planning, LLM visual prompt review, deterministic visual prompt sanitation, strict ModelsLab image generation, durable continuous-audio render, and Codex-assisted upload packaging. The full production workflow is documented in `docs/workflows/video_production_workflow.md`.

## Hard Rules

- Do not use source-seed scene annotations. Final semantic truth comes from the locked script.
- Run identity is a production gate, not an admin detail. Do not ingest, generate, render, or create new episode folders until `run_identity.json` exists and matches the command flags. Episode numbers live in `--episode` (`ep_01`, `ep_02`, etc.), not in the run/week slug. A title or operator phrase such as "Part 2" means `--episode ep_02` by default; do not create `part-2/episodes/ep_01` unless the operator explicitly approves a separate standalone run.
- Scope is a production gate too. If `run_identity.json`, the operator request, or the current instruction says proof, sample, first 20 minutes, first 3 minutes, first N cuts, or another bounded scope, do not generate, regenerate, render, or refresh the full episode. Select only the scoped cut IDs/artifacts, write proof reports with proof-specific names, and stop for operator approval before scaling to full production. A ledger stage such as `premium_render` does not override an explicitly bounded proof request.
- When a proof run validates a planner or render fix, promote the durable behavior into code, docs, and fixture tests before scaling to the full episode. Do not let proof-sidecar artifacts silently become production truth; the full episode must resume from the official `run status` stage and regenerate the official artifacts through the guarded command path unless the operator explicitly approves a scoped diagnostic bypass.
- Every run must have a visible stage ledger before work continues: current stage, required input artifact, output artifact, whether operator approval is required, and whether the artifact exists. If asked to "continue," first identify the current stage from artifacts and the ledger instead of guessing. Use `node bin/goldflow.mjs run status --channel <channel> --series <series> --week <week> --episode <episode> --format markdown` or `--episode-dir <episode-dir> --format markdown`; do not continue from memory alone.
- Codex/agent operating protocol for this repo: read the full `AGENTS.md`, run `goldflow run status`, state the current unresolved stage and next artifact, then run only the next valid workflow command unless the operator explicitly redirects. If a command would skip a missing upstream artifact, stop and explain the missing gate instead of improvising.
- The CLI workflow guard is part of production safety. Normal production commands must match the first missing `run status` stage. If `goldflow` blocks a command, do not work around it by calling scripts directly. Use `--workflow-bypass true` only for explicit operator-approved diagnostics, recovery, or one-off inspection, and state why the bypass is safe before running it.
- After every completed stage, run `goldflow run status` again, read `current_stage` and `next_command_shape`, and execute only that next command shape. Never bypass the guard just because you know the nominal order from memory.
- Source scripts should arrive as spoken narration prose only. Use `docs/prompts/manhwa_recap_chatbot_prompt_v1.md` for manhwa recap chatbot generation unless a newer documented template is selected.
- New production narration targets 210-220 spoken WPM. Run `goldflow script pace-check` after exact-hash approval and `goldflow audio pace-check` after local Whisper timing. `script_pace_report.json` must block if detected hook milestones are late, even when estimated WPM/runtime are acceptable; fix the source/chatbot hook rather than expecting visual/audio stages to save it. `run status` must block if `script_pace_report.json` or `narration_pace_report_<episode>.json` is missing, stale, blocked, or outside the 210-220 WPM actual-audio range. If Qwen TTS lands outside the audio pace gate, use the explicit recovery command `goldflow audio tempo-normalize --target-wpm 215`, then rerun local Whisper timing and audio pace check before timing bind. Do not hide manual `ffmpeg atempo` fixes outside the ledger. For explicit validation batches where the operator provides reviewed/approved scripts and says TTS WPM is diagnostic, lock `--pace-policy diagnostic` in preflight; run script pace with diagnostic hook warnings and audio pace-check with `--pace-policy diagnostic`. The reports must still record hook warnings, `actual_wpm`, `diagnostic_hook_status`, and `diagnostic_pace_status`, but they must not block production or route to source rewrites/tempo-normalize solely for diagnostics.
- Chatbot prompts are versioned in `docs/prompts/`; improve those templates before adding deterministic source-prose repair to the pipeline.
- Do not creatively rewrite approved scripts with deterministic code.
- Treat the chatbot/operator-approved story as production truth. Qwen may analyze, extract, and flag issues, but it must not be trusted to improve story prose by default.
- Generic script enhancement is optional and pre-lock only. If enhancement changes the script, every downstream artifact must be regenerated from the new exact hash.
- Keep caption text, spoken TTS text, and semantic visual facts as separate layers. Captions preserve the approved script; TTS may use approved speakable equivalents; visuals use extracted scene facts. TTS-only pronunciation overrides must never leak into subtitles, visual prompts, semantic facts, titles, descriptions, or rendered on-screen text.
- Protected terms need explicit speakability handling before TTS, especially ranks, UI labels, odds, decimals, currencies, acronyms, and system messages. Do not add broad phonetic rewrites for ordinary character names by default; the Anti-Simp run showed that phonetic spellings such as name-syllable variants can make the TTS pronounce names inconsistently and can leak into subtitles. Prefer letting the TTS read ordinary names directly unless a spot-listen proves a specific name is wrong, then add the narrowest TTS-only override and verify subtitles remain script-faithful.
- Streamer scripts need explicit TTS-only homograph handling for phrases such as `live content`, `streaming live`, `live stream`, and `stream content`. Prefer source/chatbot wording like `livestreaming`, `livestream`, `live broadcast`, `stream videos`, or `livestream video content`; if the approved script still contains risky wording, targeted speakability may add narrow `qwen_spoken_text` replacements without changing captions or `script_clean.md`.
- Targeted script speakability is the preferred TTS guidance stage. It must not mutate `script_clean.md`; it writes `script_speakability_report.json`, `tts_spoken_overrides.json`, and `script_speakability_problem_areas_report.json`. It should fix known problem areas only, such as explicit narrator self-reference ("the narrator wants you to understand") that sounds artificial when spoken. Broad speakability can introduce unnatural rewrites and should only run on explicit operator request.
- Do not run SFX or score planning before final narration exists.
- Do not run SFX or score planning without current local Whisper timing.
- Segment or Qwen timing is fallback metadata only; production SFX/score plans must be stamped with `timing_source: "local_whisper_word_timing"`.
- Default preflight audio target is `narrator_only`. For narrator-only runs, `run status` must skip `sfx_score_plan` and point the next audio command to `audio longform-bed --narration-only true`. Do not run audio enrichment, score generation, SFX bank resolution, or transition SFX unless the operator explicitly opts into an audio-design variant.
- Planning LLM routes are Codex or local Qwen only. Do not use ModelsLab LLM endpoints for semantic, audio, visual reference, visual prompt, or prompt review planning. ModelsLab is a media generation provider in this workflow, not a planning backend.
- Image provider routing is explicit and locked at preflight. If the operator says "run it through the pipeline normally" or does not name an image provider, set `--image-provider modelslab` in `goldflow run preflight` and use the goldflow default image lane: ModelsLab Flux Klein for references and scene image cuts, with Codex/local-Qwen used only for LLM planning/review tasks. `imagegen start` must refuse a provider that differs from `run_identity.json` unless the operator explicitly approves `--confirm-image-provider true`. If the operator says "Codex Imagen", "Codex imagegen", "use Codex for images", or similar, lock that in preflight and do not run the ModelsLab route for scene cuts. If the operator explicitly requests the speed-test hybrid lane, lock `--image-provider hybrid_codex_refs_multichar`: imagegen routes all references through Codex imagegen, routes risky multi-character cuts through Codex imagegen, and routes lower-risk simple scene cuts through ModelsLab. If the operator requests Codex for the retention opening only and wants references through Codex too, lock `--image-provider hybrid_codex_opening_modelslab_rest` and persist the window with `--codex-opening-sec <seconds>` in `run_identity.json`: all references use Codex imagegen, scene cuts before that timestamp use Codex, and all later scene cuts use ModelsLab. If the operator requests ModelsLab references plus Codex opening scene cuts, lock `--image-provider hybrid_modelslab_refs_codex_opening_modelslab_rest --codex-opening-sec <seconds>`: references are generated by ModelsLab, scene cuts before the locked timestamp are staged/imported from built-in Codex imagegen workers, and later scene cuts are generated by ModelsLab with `--provider-filter modelslab`. Hybrid lanes are experimental provider lanes, not the normal default.
- For narrator-only runs, `sfx_score_plan` is the skipped stage. `audio longform-bed --narration-only true` is still required and is the final audio bed consumed by render; do not skip it.
- Codex Imagen means the built-in Codex image generation tool path, executed by Codex subagents/workers for concurrency. The production route is: assign scoped cut IDs to workers, have each worker use built-in Codex image generation to create a real raster, save exactly one PNG named for the assigned image ID inside an isolated staging folder, report the SHA-256, then import with `goldflow imagegen import-staged-codex`. Do not use nested `codex exec --enable image_generation` as the production Codex Imagen route; if it returns JSON/text such as `{"status":"success"}` without a raster, that is a failed provider probe. Do not copy the newest/global `CODEX_HOME/generated_images` file, do not mix stale staged images, and do not fall back to ModelsLab unless the operator explicitly approves that fallback in the current run.
- ModelsLab image generation is resumable by design. After a partial failure, rerun the same `imagegen start` command or retry only the failed cut ids at lower concurrency. Never use `--force` unless intentionally replacing a known-bad image, never rerun upstream visual planning/review/harden between retries for the same cuts, and never restart the pipeline from an earlier stage to recover image failures.

- SFX-only is an approved production route for narration-driven humiliation/system stories. Run audio enrichment with `--sfx-only true`, generate no score beds, and mix with `--skip-score true` so the final audio is narrator plus selected SFX only.
- SFX generation prompts must describe the concrete sound source, material, action, space, duration feel, and intensity. Do not send story summaries as SFX prompts. Good prompt shape: "short cold crystalline digital transaction chime, quick attack, clean decay, subtle corporate interface texture, no melody, no speech." For crowd/room cues, request nonverbal texture such as a brief wealthy-room laugh ripple, shocked gasp wave, applause swell, or sudden room hush.
- SFX should be selective and beat-anchored. Prioritize system pings, phone buzzes, paper/card handling, glass/banquet objects, room hushes, laugh ripples, applause turns, wealth-reveal impacts, and a few signature transaction confirmations. Avoid constant literal Foley under every sentence.
- Maintain the SFX bank as a curated production asset library, not a raw dump. Run `goldflow sfx-bank rebuild` after legacy/generated assets are added, then `goldflow sfx-bank audit` before reuse-heavy runs. Use `goldflow sfx-bank list --status needs_review --query <term>` to find candidates for spot-listen. Reject bad assets with `goldflow sfx-bank reject --cue <cue_id> [--asset <asset_id>]` and promote good alternates with `goldflow sfx-bank prefer --cue <cue_id> --asset <asset_id>`. The enrichment planner should reuse tight bank matches only when the preferred asset is explicitly `available` and the cue intent matches. Do not keep regenerating common edit sounds such as whoosh-pop, manga-panel snap, system ping, room hush, crowd gasp, camera flash pop, screen-scan zip, and low sub impact once a good banked asset exists.
- Generated SFX assets with vague or abstract prompts must be treated as review candidates, not defaults. Cues named like `incidental_<line text>`, or prompts built around words such as "bloom", "pressure", "energy", "aura", "truth", "destiny", "void", "worship", or "reveal tone" should be rewritten into concrete source/action prompts or rejected after spot-listen.
- Score/music generation can use local ACE-Step 1.5 by setting `ANIFACTORY_SCORE_PROVIDER=local_ace_step` or passing `--score-provider local_ace_step`. This feeds each beat-mapped chapter `ace_step_prompt` to `/Users/joel/AniFactoryTools/ACE-Step-1.5` and writes beds under `assets/audio/ace_step_score_beds`. Default local model selection is DiT `acestep-v15-turbo` with LM `acestep-5Hz-lm-1.7B`; override with `ANIFACTORY_ACE_STEP_CONFIG_PATH` and `ANIFACTORY_ACE_STEP_LM_MODEL`.
- Score should be moment-directed, not automatic. For runs where the story is stronger without continuous music, use `--score-mode drops_only`: `score_chapter_plan.json` stays empty and `score_drop_plan_<episode>.json` places Whisper-timed local ACE-Step scoring cues only on focal drama, hype, reversal, reveal, payoff, escape, and cliffhanger beats. The longform mixer fades each drop in/out; if chapter beds exist, it ducks overlapping beds instead of stacking uncontrolled volume. Target roughly 20-35 score drops by default, but explicit "scoring team" or loud retention runs should use `--retention-mix true` or raise the cap with `--score-target-drops-max`/`--score-target-drops` and hold major moments for 8-18 seconds. For "sense/loud/final mix" retention runs, it is acceptable to push to roughly 60-80 score drops with frequent dramatic score punctuation. Keep score separate from physical SFX and intentional enough that each cue feels placed.
- When only the score/drop layer needs improvement, use the chunked score-only planner (`goldflow audio score-drops-chunked`) instead of rerunning the full SFX/score enrichment prompt. The full enrichment prompt is rich enough to stall on long episodes; chunked score planning must consume current Qwen segment timing plus local Whisper timing, write production-compatible `score_chapter_plan.json` and `score_drop_plan_<episode>.json`, and leave existing narrative SFX/ambience plans untouched.
- ACE-Step score drops are currently generated episode-local under `assets/audio/ace_step_score_drops`. Treat them as reusable only after spot-listen and deliberate promotion into a curated score-drop bank/preset list. Do not assume every generated drop is a good reusable asset just because the technical generation passed.
- Maintain a score-drop bank separately from the SFX bank. Run `goldflow score-bank rebuild` to index episode-local ACE-Step drop metadata, `goldflow score-bank audit` to see review status, `goldflow score-bank list --status needs_review --query <tag>` for spot-listen batches, then `goldflow score-bank approve --id <bank_id>` or `goldflow score-bank reject --id <bank_id> --note <reason>`. Only approved score-bank drops should become reusable presets; normal episode score generation still uses fresh ACE-Step drops planned from the current story.
- SFX should be dense in the opening hook. For the first 30 seconds, target at least 14 and ideally 18-24 audible cues where narration and edit timing support it: deterministic transition accents such as swipe-up flash, swipe-down whoosh, hard scene-card whoosh, impact flash, dark-paper title snap, and manga-panel slide should be bound by the visual transition plan and render layer to actual cut timestamps, while LLM-authored narrative SFX should cover system/ledger pulses, room hushes, phone buzzes, transaction chimes, shame hushes, and concrete story/object sounds. After the hook, keep SFX consistently present but selective and beat-anchored. The Anti-Simp dense-audio pass used a successful target shape of about 260 narrative SFX/ambience cues plus deterministic transition accents and 78 score drops for a 62-minute episode; treat that as a loud retention reference point, not a mandatory count for every runtime.
- Ambience is generated as loopable SFX, not as score. Use low nonmusical environmental beds between score drops for the current script's actual locations: campus hallway air, party room tone, rain street, bedroom/dorm tone, cafeteria crowd bed, stadium crowd air, digital system hum, corporate room tone, event-stage air, office hum, server room, vehicle cabin, rooftop air, exterior wind, etc. Ambience prompts must avoid melody, rhythm, vocals, speech, or crowd dialogue and should sit around -34 to -28 dB unless the operator asks for a louder cinematic final mix.
- Loud retention mixes should not use the old quiet `--sfx-boost-db -4` setting. Start with `--score-drop-boost-db 3` to `5`, `--signature-sfx-boost-db 1` to `3`, `--incidental-sfx-boost-db 1` to `3`, and `--ambience-sfx-boost-db -2` to `0`, then verify against narration intelligibility. Bad-sounding SFX must be regenerated from concrete prompts, not hidden by lowering the whole SFX layer.
- ModelsLab score generation remains available with `ANIFACTORY_SCORE_PROVIDER=modelslab` and uses `/api/v6/voice/music_gen` model_id `ai-music-generator`.
- Narrator-only is the default voice route. Character voice casting requires an explicit operator request and flag.
- Ambiguous dialogue routes to narrator.
- Render must consume one continuous final mixed audio track.
- Qwen TTS stitch must include a small inter-segment safety gap by default so unit boundaries do not clip final phonemes or smash narration beats together. Current default is `ANIFACTORY_MODELSLAB_QWEN_SEGMENT_GAP_SEC=0.22`; rerun Whisper after any stitch or tempo-normalization change.
- Narration loudness is controlled at longform mix, not by destructively editing cached TTS assets. Production starting point is `--narration-volume-db 2` with the final limiter enabled. For retention/final review mixes where the narrator feels behind SFX/score, use the corrected narrator-forward recipe: `--narration-volume-db 3 --score-drop-boost-db 3 --signature-sfx-boost-db 2 --incidental-sfx-boost-db 2 --ambience-sfx-boost-db -2 --target-lufs -13 --true-peak-db -1`, then verify with `ffprobe` and `ffmpeg ... -af volumedetect`. If narration still competes with music, lower score beds or drops before pushing narration harder. If the mixer lands low after AAC encode, run a final loudnorm corrective encode/remux rather than shipping a quiet file.
- Visual planning must use current-scene facts only. Do not import neighboring context, stale refs, standalone negative-prompt payloads, or characters not visible in the scene.
- Visual reference scoping invariant: `style` is the only global reference kind. Location, character, prop, UI, action, and effect refs must be scene-scoped and attached only from the current scene candidate set. Deterministic code may validate, drop, and block out-of-scope refs; it must not guess replacement refs or rewrite scene content.
- Semantic `ref_requirements` create scoped reference targets; they do not automatically require standalone image generation. Keep standalone refs for recurring characters, major character states, opening-retention location anchors, key recurring locations, signature recurring system/UI motifs, critical recurring props, and high-risk physical-contact one-scene characters. Downgrade one-scene late locations/UI/props to `no_ref_needed`; downgrade minor recurring late locations/UI/props/actions to `derive_from_best_cut` or `derive_from_first_clean_cut` when an initial clean scene image can become the reference later. Do not generate standalone refs for minor role characters, generic witnesses/crowds, single-use wardrobe variants, one-off dashboards, one-off documents, or 2-3 occurrence minor assets unless they are truly critical to the story. If the style bible/text style prompt is sufficient, drop generated style refs for the run.
- Derived reference targets are a resumable imagegen subflow, not a reason to rerun visual planning. Targets with `generation_mode` such as `derive_from_first_clean_cut`, `derive_from_first_clean_wide_cut`, or `derive_from_best_cut` stay pending until a real scene cut image is generated, then `goldflow imagegen promote-derived-refs` copies the chosen cut into `assets/images/references` and updates the reference artifacts. After promotion, `imagegen start` attaches the now-real reference from the hardened prompt manifest at runtime; do not rerun `visual plan`, `visual review`, or `visual harden` just to pick up the promoted path.
- Visual reference planning derives location target `scene_ids` from semantic `ref_requirements` and unions them with LLM-authored ids. Any physical scene that requires a location ref but has no covering target must block with `scene_missing_location_ref`. The blocker means the scoped target is missing, not that the target must be generated as a standalone image.
- Visual review should run with `--auto-resolve true --max-resolve-iterations 2` by default. Auto-resolve replans exact blocked `image_id` cuts when every blocker names an image id; it falls back to blocked `scene_id` scope only for true scene-level blockers. It uses specific correction directives, re-reviews regenerated cuts/scenes only, preserves untouched image/scene/visual beat identity, and writes `visual_resolution_deadletter.json` with status `blocked_deadletter` when regular capped failures remain. If all remaining capped blockers came from compatible harden feedback, write `visual_manual_agent_review_<episode>.json` and hold the reviewed plan at `needs_manual_agent_review`; a Codex/operator review must decide whether the prompt/ref plan is wrong or the harden detector is too strict before production continues. If `section_image_prompts.json` already exists and `section_image_prompts_reviewed.json` is blocked or harden reported compatible blockers, resume with `visual review --resume-blocked true --auto-resolve true --max-resolve-iterations 2`; do not rerun full `visual plan` unless the plan artifact is missing, stale, corrupt, or the operator explicitly approves a full replan. `run status` must surface the smallest next substep inside the visual plan/review/harden block: `visual plan` only when the prompt plan is missing or invalid, `visual review` when the plan exists, resume-review when reviewed prompts are blocked, manual agent review when capped harden feedback is ambiguous, and `visual harden` when review has passed. If the previous guarded attempt reached `visual harden` and produced compatible blockers for the same episode/script, the next review auto-resolve pass must feed those harden blockers back to the LLM and validate repaired cuts/scenes with harden inside the resolve loop. Imagegen must refuse dead-lettered or manual-review-held scenes.
- The LLM is the creative visual author. Give it the story chunk, premise/bible context, approved refs, state contracts, current beat excerpt, and neighboring beat summaries needed to make the same character, location, composition, and ref decisions a hands-on agent would make. Deterministic production code must not creatively choose scene content, infer missing locations, add characters, rewrite action, or patch narrative intent.
- Visual prompt planning must consume `visual_beat_plan.json` when present. Semantic scenes are not image cuts; long scenes must be split into editorial visual beats before imagegen. Beat planning is transcript-first, not duration-first: every beat needs an exact local final-script excerpt, Whisper-derived start/end timing, editorial cues, `visual_job`, `suggested_shot_job`, local location, visible characters, mentioned-only characters, props/UI, and beat-level advisory `ref_needs` or `beat_ref_requirements` with generation-mode hints. The planner asks what the viewer needs to see now to understand, feel, and keep watching; it does not merely divide scenes into equal-duration chunks.
- Editorial beat jobs are mandatory. Each cut should have a purpose such as premise image, humiliation image, system/quest reveal, reaction shot, remote witness cutaway, chat/UI insert, location transition, physical entrance, consequence, status turn, payoff, or cliffhanger question. Split on story turns: new location, named character entry, system message, public-audience/status-metric change, emotional pivot, objective change, threat reveal, public witness reaction, and hard scene transition.
- The first 3 minutes are the retention runway. Treat 0-30 seconds as the dense cold open and 30-180 seconds as the retention ramp. For any story with an early hidden power, public humiliation, ranking/status rule, regression, system, or major premise reversal, aim for the title promise by 0:30, the first visible proof/payoff by about 0:45, the core status or power rule explained before or around that payoff, and the next major arc underway by about 1:00 when the script supports it. Visual beat planning should target roughly 8-12 distinct opening cuts with `--hook-duration-sec 30 --hook-target-beat-sec 3.2 --hook-max-beat-sec 4.2`, then keep 30-180 seconds faster than normal with `--retention-ramp-sec 180 --ramp-target-beat-sec 5.2 --ramp-max-beat-sec 6.5`. Do not let the first 3 minutes sit on slow Ken Burns images while audio drops are firing.
- After the first 20 minutes, visual pacing may relax when retention is already established. Favor longer intentional holds, reused approved images, and slower cut turnover for stable dialogue, analysis, aftermath, or low-risk explanation beats, while still generating fresh images for new locations, character state changes, reveals, public status turns, action, emotional pivots, and cliffhangers. Do not slow the opening hook or early retention runway just to save image spend.

- Character aliasing and role labels must be resolved from approved character_state_refs before imagegen. A role label attaches a character ref only when that person is a visible subject, especially in multi-character shots.

Source images supply facial likeness only; the character-state ref supplies wardrobe, pose, body state, and episode styling.

- Memory/reflection beats should not put faces inside props. Stage memories as separate translucent silhouettes in open air near the object, with the object itself kept clean and readable.

The current strongest recipes are humiliation and simp framing: "wanted to humiliate me", "world's biggest simp", "my system awakened", "regressed", "left me for boss/rich man", and concrete sacrifice/debt proof. Titles should pair a simple humiliation/simp premise with a clear reversal. Thumbnail overlays should use short bubble-letter labels, yellow bubble arrows with white glow and black outline, and mobile-readable words such as `SIMP`, `LOSER`, `SYSTEM ACTIVATED`, `HUMILIATED`, `REGRESSED`, or a single concrete debt/amount when story-critical.

## Current Production Models And Methods

- Source: polished narration prose from the operator/chatbot, with exact-hash script approval.
- Speakability: targeted-only by default; broad speakability is opt-in.
- Voice: narrator-only default through ModelsLab Qwen TTS using the locked Joel narrator clone.
- Timing: local Whisper word timing is production truth for subtitles, SFX, score, visual beats, and render.
- Audio planning: Codex or local Qwen only. If automated planner calls are unavailable, a Codex-agent manual plan may be written with explicit provenance, current source hashes, and Whisper timing.
- SFX generation: ModelsLab `/api/v7/voice/sound-generation` assets are allowed after a Codex/local-Qwen/agent-authored plan. SFX events must use locked asset paths before mix.
- Ambience generation: also ModelsLab sound-generation, represented as loopable SFX events with `recurrence_class: "ambience"`, `loop: true`, short generated `asset_duration_sec`, and longer timeline `duration_sec`.
- Score generation: prefer local ACE-Step 1.5 for moment score drops and any intentionally chosen chapter beds; current default local model pair is `acestep-v15-turbo` plus `acestep-5Hz-lm-1.7B`. Score drops are short Whisper-timed ACE-Step riser/hit accents for 20-35 drama, hype, reveal, and payoff beats, mixed by ducking the base score bed if one exists.
- SFX-only mix: use `--sfx-only true` during audio enrichment and `--skip-score true` during longform-bed when the story is stronger with clean narration and punctuation SFX instead of score.
- Default image generation: ModelsLab Flux Klein, with normal production prompts, no standalone negative-prompt payloads, and explicit reference slot mapping. Explicit Codex/OpenAI image requests stay on the Codex/OpenAI lane and require real-raster verification before any batch spend. Explicit hybrid speed-test requests use `--image-provider hybrid_codex_refs_multichar`, where references and risky multi-character cuts use Codex imagegen while simpler cuts use ModelsLab. Explicit retention-window hybrid requests use `--image-provider hybrid_codex_opening_modelslab_rest --codex-opening-sec <seconds>`, where all references use Codex imagegen, scene cuts before the locked timestamp use Codex, and the rest of the video uses ModelsLab.
- Render: one continuous mixed audio track, final-script subtitle text timed by Whisper, yellow subtitle text with a small black outline and no background box. Premium full-frame profile-based Ken Burns remains the default production render profile: `--motion fill_ken_burns --motion-strength 1.75 --render-scale-multiplier 1.45 --render-concurrency 4 --clip-preset veryfast --final-preset veryfast`. The official no-oversample smoother sibling for testing is `--motion smooth_fast_ken_burns --motion-strength 1.75 --render-concurrency 4 --clip-preset veryfast --final-preset veryfast`; it defaults to 60 fps and render-scale multiplier `1.05`, keeps the same intentional hook/action/reveal/wide/emotional movement grammar, but removes long calm micro-drifts and softens sharpening on long holds to reduce mid-video shake. Use blurred foreground drift or `fill_pan` only as an explicitly approved diagnostic fallback after these profile-based renders are proven unsuitable for the specific episode. Motion should be visible and intentional: hook bursts, action pushes, reveal pushes, wide drifts, emotional holds, steady pushes, lateral trucks, vertical/diagonal moves, and true zoom-outs that expose more of the image canvas on establishing or reveal beats. A premium render must not be all push-ins under different profile labels; check `render_motion.motion_behaviors`, `render_motion.mode`, `render_motion.fps`, and `render_motion.render_scale_multiplier` before calling the MP4 review-ready.
- Render timing is absolute. Preserve the `start_sec` timeline from `section_image_prompts_hardened.json` / `visual_beat_plan.json`; do not globally scale image durations to fill the audio. If the visual plan has a gap before the next cut, hold the current image until the next cut's real start time or add new visual beats/images in a planning pass. Never fix timing drift by stretching all cuts. Each generated motion clip must be duration-probed immediately; abort on malformed clips instead of continuing a long render.
- Render polish should keep improving from the premium default: add better beat-selected FFmpeg `xfade` transitions, manga-panel swipes, tasteful flash/sweep overlays, and transition SFX bound to the actual visual cut timestamp from the transition edit plan. Do not ask the audio SFX planner to author generic edit whooshes/pops as narrative SFX. The visual transition planner decides which cut boundaries deserve transition SFX and which xfade family fits; render/edit applies those SFX at the cut. Do not default every scene change to fade-to-black, and do not let transition work break absolute cut timing.
- Opening retention QA must verify three layers together: visual cuts, transition SFX, and score drops. In the first 30 seconds, the render should have a visible cut every 2-4 seconds when narration supports it. From 30-180 seconds, cuts should still arrive often enough to feel hand edited, usually every 4-7 seconds when the story is actively escalating. Transition SFX should start within roughly one frame of selected visual cuts, and score drops should land on hook/ramp turns that need sustained emotion or threat. If the audio is active but the visuals barely change, regenerate/replan first-3-minutes visual beats before the full render.
- Render QA is mandatory before treating an MP4 as publishable: run `ffprobe` for duration, fps, codec, pixel format, audio codec, and size; audit rendered cut times against transcript/visual-beat `start_sec`; extract final QA frames/contact sheets; and spot-check prompt/image/ref consistency. A final MP4 with the right total duration is not enough if individual images drift from the story.

## Next Run Policy

- Treat the operator/chatbot-provided polished narration story as the candidate source of truth.
- Do not run broad enhancement by default. Use targeted speakability and readiness checks after approval, and only apply TTS-only overrides or explicitly approved script edits.
- Use narrator-only voice unless the operator explicitly requests character casting.
- Use local Whisper word timing before SFX, score, visual beats, engagement overlays, subtitles, or render.
- Use local ACE-Step for production score beds. ModelsLab music generation is a fallback only when explicitly requested.
- Use the optional score-drop layer when the run needs retention accents. Normal runs can use twenty to thirty-five Whisper-timed local ACE-Step riser/hit accents; loud retention/final-review runs should use chunked score planning around 60-80 held drops with 8-18 second durations when the operator asks for more score punctuation.
- For humiliation/system-finance stories, consider an SFX-only audio pass before committing to score. Clean narration plus selective SFX may retain better than continuous beds.
- For imagegen, prioritize reference quality and spot checks over raw throughput. Start concurrency around 6-12 on the next full run, then raise only after references and prompt quality look stable.
- Babysit the next run through staged visual gates before trusting full automation: manually review references, review the sanitized prompt sample, generate and contact-sheet the first small image batches, then keep periodic contact-sheet spot checks through high-risk sections. Do not treat imagegen as set-and-forget until several consecutive batches show no duplicate protagonist, subject fusion, destructive UI, wrong refs, or major location/style drift.
- If generation times out, count missing references/cuts and resume only those IDs with cache reuse. Do not force-regenerate completed good images.
- Keep narration competitively loud from the longform mix using `--narration-volume-db 2` as a safe baseline, then verify clipping and intelligibility against score beds and drops. When the operator asks for the loud retention/final review version, use the narrator-forward recipe (`--narration-volume-db 3`, score-drop boost around `3`, target `-13 LUFS`, true peak `-1 dBTP`), dense opening narrative SFX, ambience where it fits, and moment-directed score drops rather than an always-on score bed. Transition SFX are not part of audio planning by default; the visual transition plan and render/edit layer place them on selected cuts.
