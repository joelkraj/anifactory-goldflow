# AniFactory Goldflow

This repo is the clean longform production lane. It intentionally excludes legacy render paths, old visual heuristics, source-seed annotations, regex SFX production fallback, and multi-route compatibility behavior from the original AniFactory repo.

## Non-Negotiable Flow

Production moves through one artifact chain. Before step 0, run identity preflight is mandatory: decide the channel, stable series slug, stable run/week slug, exact episode id, title, image provider, audio target, and proof-vs-production intent, then write `run_identity.json` with `goldflow run preflight`.

0. Generate or obtain a polished narration story using the source-script workflow in `docs/workflows/source_script_generation_workflow.md`.
1. Ingest a polished narration story into `script_clean.md`.
2. Optional script polish/enhancement only when the operator explicitly asks for it, before approval, with the resulting script treated as a new candidate for review.
3. Manual review and operator approval for the exact final script hash.
4. Script pace check for the approved script. New production scripts target 195-220 spoken WPM; `script_pace_report.json` must exist for the exact approved hash before speakability or semantic planning.
5. Targeted production-readiness and speakability QA from the approved script. These passes may suggest TTS-only fixes for known problem areas, pronunciation mappings, and risk notes, but must not silently rewrite the story. Broad speakability is optional and must be explicitly requested.
6. Semantic continuity extraction from the locked script and bibles: overlapping chunks run concurrently (four workers by default), then one global evidence-constrained reconciliation canonicalizes aliases, characters, locations, props, UI, and state transitions into `story_fact_ledger.json` without inventing facts.
7. Narrator-only voice plan by default. Voice planning requires current `script_speakability_report.json` and `tts_spoken_overrides.json` unless explicitly run in diagnostic bypass mode.
8. ModelsLab Qwen TTS and stitch at the provider-native speed locked in `run_identity.json` (default `1.25`). Native TTS delivery is the production pace control; post-TTS tempo processing is not part of the normal lane.
9. Local Whisper word timing on the final stitched narration.
10. Audio pace check from Whisper timing. `narration_pace_report_<episode>.json` must pass at 195-220 actual WPM before timing bind or downstream production. If it misses, regenerate Qwen at an adjusted provider-native speed and rerun Whisper. `audio tempo-normalize` is emergency-only and requires explicit operator approval.
11. Review TTS and timing artifacts before downstream production.
12. Default narrator-only longform audio bed from the stitched narration/Qwen report. SFX, score, ambience, and transition SFX are opt-in variants, not default requirements.
13. Visual beat planning: atomize the exact final narration into clause/sentence units bound to contiguous Whisper word indices, then let the LLM editorial director regroup adjacent atoms. The LLM owns visual job, depiction mode, physical/screen/preview/mentioned entities, location, action, and composition. Deterministic validation owns complete word coverage, order, transition barriers, stable span-derived IDs, evidence membership, retention timing rails, and cumulative `active_state_constraints`. Semantic scenes remain broad story truth; local atoms and the fact ledger are cut truth. Beat-level ref hints are evidence only and never automatically promote a story cut into a reference.
14. Visual reference director planning: first write broad non-authoring evidence to `reference_evidence_ledger.json` and textual physical-location truth to `location_contract_ledger.json`. Chunk LLM calls may propose candidates, but one global reference-director LLM is the sole creative selector. Its final clean attachable selections become the lean prompt-facing `visual_reference_plan.json`, `character_state_refs.json`, and selected-only `reference_inventory_ledger.json`. Deterministic code may canonicalize, union explicit contract scene scope, validate, and add approved source-only dependencies; it must never restore or invent an omitted generated target.
15. Review and hash-approve the selected reference plan with `goldflow visual approve-ref-plan` before any reference-generation spend. Each selected conditioning asset must contain exactly one concept: one identity/state, environment, prop, UI motif, faction/uniform language, action/effect language, or abstract style language.
16. Reference generation in dependency order: style ref first only when needed, then clean character/location/prop/UI/faction/action anchors.
17. Separately approve the generated reference images with `goldflow visual approve-refs` before visual prompt planning. Do not hand-edit `character_state_refs.json` approval state.
18. Visual prompt planning consumes approved references, current-scene facts, and visual beats.
19. Visual prompt harden runs directly on `section_image_prompts.json`. It validates LLM-authored prompt/ref manifests, resolves approved ref paths, strips forbidden or unknown refs, enforces the four-reference cap, normalizes manifest/reference structure plus format-only prompt syntax, and writes a pre-imagegen sample sheet. It does not rewrite depicted prompt content.
20. Visual prompt review is not a default full-episode toll. If harden finds compatible creative blockers, run `visual review --blockers-only true --auto-resolve true --max-resolve-iterations 2` so only named cut IDs are repaired, then harden the reviewed artifact. Full review remains an explicit diagnostic mode and must write `review_value_report_<episode>.json`.
21. Visual transition planning with silent transition SFX by default.
22. Image generation with a representative provider probe for premium routes, a provider circuit breaker, resumable cut-scoped retries, and `cut_execution_ledger.json` hash tracking.
23. Image output QA before render. Build ordered five-minute contact sheets, inspect the first three minutes plus physical-action, dense-cast, four-reference, and explicitly risky cuts, reject only failed cut IDs, and write a passed `image_output_qa_<episode>.json`. Generated-image spelling is outside this QA contract and must not trigger regeneration.
24. Render with intentional beat-selected transitions, phrase-aware final-script subtitles timed to Whisper, one continuous mixed audio track, hash-cached motion clips, and the smooth-fast Ken Burns profile by default.
25. Final QA: spot-listen narration cuts, spot-check prompt/image/ref consistency, verify transition style, verify subtitles are final-script text timed to Whisper, then `ffprobe` the final upload-safe render.
26. Package for upload: title, thumbnail, and description hooks should be generated after the story/render is understood, using proven channel packaging recipes and current episode truth.

Current migrated scope is source ingest, script approval, targeted speakability, semantic scene planning, the audio spine, Whisper timing, timing binding, SFX/score enrichment, longform audio mix, visual beat planning, visual reference planning, current-scene-only visual prompt planning, LLM visual prompt review, deterministic visual prompt sanitation, strict ModelsLab image generation, durable continuous-audio render, and Codex-assisted upload packaging. The full production workflow is documented in `docs/workflows/video_production_workflow.md`.

## Hard Rules

- Do not use source-seed scene annotations. Final semantic truth comes from the locked script.
- Run identity is a production gate, not an admin detail. Do not ingest, generate, render, or create new episode folders until `run_identity.json` exists and matches the command flags. Episode numbers live in `--episode` (`ep_01`, `ep_02`, etc.), not in the run/week slug. A title or operator phrase such as "Part 2" means `--episode ep_02` by default; do not create `part-2/episodes/ep_01` unless the operator explicitly approves a separate standalone run.
- New runs use `goldflow_run_identity_v2` and lock the Git commit, branch, dirty-diff hash, stage-registry version, provider/model versions, and proof scope. Production preflight requires a clean worktree. A dirty worktree may be recorded only for an explicit diagnostic/proof with both `--allow-dirty-worktree true` and `--dirty-reason <reason>`; it is never a production shortcut.
- Scope is a production gate too. If `run_identity.json`, the operator request, or the current instruction says proof, sample, first 20 minutes, first 3 minutes, first N cuts, or another bounded scope, do not generate, regenerate, render, or refresh the full episode. Select only the scoped cut IDs/artifacts, write proof reports with proof-specific names, and stop for operator approval before scaling to full production. A ledger stage such as `premium_render` does not override an explicitly bounded proof request.
- A bounded proof may reuse an audited narration/Whisper baseline only through `goldflow run import-proof-baseline --baseline-episode-dir <dir>`. The command is proof-v2-only: it trims the locked audio window, re-hashes it, scopes and re-times Whisper words, records all baseline hashes, emits zero-cost imported voice/TTS artifacts, and cannot run for production identities. When semantic planning precedes that import, pass the audited timing with `semantic plan --proof-baseline-word-timing <path> --scope-start-sec <n> --scope-end-sec <n>` so semantic extraction does not analyze the full episode.
- When a proof run validates a planner or render fix, promote the durable behavior into code, docs, and fixture tests before scaling to the full episode. Do not let proof-sidecar artifacts silently become production truth; the full episode must resume from the official `run status` stage and regenerate the official artifacts through the guarded command path unless the operator explicitly approves a scoped diagnostic bypass.
- Every run must have a visible stage ledger before work continues: current stage, required input artifact, output artifact, whether operator approval is required, and whether the artifact exists. If asked to "continue," first identify the current stage from artifacts and the ledger instead of guessing. Use `node bin/goldflow.mjs run status --channel <channel> --series <series> --week <week> --episode <episode> --format markdown` or `--episode-dir <episode-dir> --format markdown`; do not continue from memory alone.
- Blockers are not automatic rerun orders. For any blocked or missing gate anywhere in the pipeline, the active agent must inspect the blocker evidence first and choose the narrowest valid recovery: a manual structured-artifact repair, a scoped rerun of only affected units, a recorded waiver/bypass with evidence, or an operator hold. Manual repairs/waivers must be written to the relevant stage artifact or a `manual_blocker_triage_<stage>_<episode>.json`/stage-specific triage artifact, must explain the evidence reviewed, and must be followed by `goldflow run status`. This flexibility does not allow skipping upstream approvals, hiding missing production assets, silently changing approved creative content, or bypassing spend/scope gates.
- Codex/agent operating protocol for this repo: read the full `AGENTS.md`, run `goldflow run status`, state the current unresolved stage and next artifact, then run only the next valid workflow command unless the operator explicitly redirects. If a command would skip a missing upstream artifact, stop and explain the missing gate instead of improvising.
- The CLI workflow guard is part of production safety. Normal production commands must match the first missing `run status` stage. If `goldflow` blocks a command, do not work around it by calling scripts directly. Use `--workflow-bypass true` only for explicit operator-approved diagnostics, recovery, or one-off inspection, and state why the bypass is safe before running it.
- After every completed stage, run `goldflow run status` again, read `current_stage` and `next_command_shape`, and execute only that next command shape. Never bypass the guard just because you know the nominal order from memory.
- Guarded stage execution is append-only. Every stage/scoped retry writes `execution_events.jsonl` plus an immutable timestamped report under `reports/stages/`; `production_manifest.json` is materialized from those events and `cut_execution_ledger.json`. Never replace historical batch cost/latency with the latest retry. `current_batch_status` and complete episode status are separate truths.
- Source scripts should arrive as spoken narration prose only. Use `docs/prompts/manhwa_recap_chatbot_prompt_v1.md` for manhwa recap chatbot generation unless a newer documented template is selected.
- New production narration targets 195-220 spoken WPM. Run `goldflow script pace-check` after exact-hash approval and `goldflow audio pace-check` after local Whisper timing. Preflight locks `qwen_native_speed` (default `1.25`) and Qwen sends that value through the provider's native `speed` parameter. Deterministic script pace checks must not contain built-in story-family, episode-specific, or premise-specific hook phrases. `script_pace_report.json` may block late hook milestones only when they come from an explicit operator/run-supplied `--hook-milestones` config; otherwise it must record that hook milestones are not configured and must not block on inferred story content. If configured hook milestones are late, fix the source/chatbot hook rather than expecting visual/audio stages to save it. `run status` must block if `script_pace_report.json` or `narration_pace_report_<episode>.json` is missing, stale, blocked, or outside the 195-220 WPM actual-audio range. If Qwen misses the audio pace gate, `run status` calculates a bounded provider-native speed adjustment and routes back to `tts qwen --native-speed <speed> --force true`, followed by fresh Whisper timing and pace check. Do not use post-TTS `ffmpeg atempo` as routine production recovery. `audio tempo-normalize` requires `--operator-approved-emergency true` and exists only for an explicit emergency diagnostic; any use requires fresh Whisper timing afterward. For explicit validation batches where the operator provides reviewed/approved scripts and says TTS WPM is diagnostic, lock `--pace-policy diagnostic` in preflight; the reports must still record configured hook warnings, `actual_wpm`, `diagnostic_hook_status`, and `diagnostic_pace_status`, but they must not block production solely for diagnostics.
- Chatbot prompts are versioned in `docs/prompts/`; improve those templates before adding deterministic source-prose repair to the pipeline.
- Do not creatively rewrite approved scripts with deterministic code.
- Treat the chatbot/operator-approved story as production truth. Qwen may analyze, extract, and flag issues, but it must not be trusted to improve story prose by default.
- Generic script enhancement is optional and pre-lock only. If enhancement changes the script, every downstream artifact must be regenerated from the new exact hash.
- Keep caption text, spoken TTS text, and semantic visual facts as separate layers. Captions preserve the approved script; TTS may use approved speakable equivalents; visuals use extracted scene facts. TTS-only pronunciation overrides must never leak into subtitles, visual prompts, semantic facts, titles, descriptions, or rendered on-screen text.
- Protected terms need explicit speakability handling before TTS, especially ranks, UI labels, odds, decimals, currencies, acronyms, and system messages. Do not add broad phonetic rewrites for ordinary character names by default; the Anti-Simp run showed that phonetic spellings such as name-syllable variants can make the TTS pronounce names inconsistently and can leak into subtitles. Prefer letting the TTS read ordinary names directly unless a spot-listen proves a specific name is wrong, then add the narrowest TTS-only override and verify subtitles remain script-faithful.
- Standalone bracketed system/UI dialogue is spoken narration, not disposable direction. Voice planning must preserve every such line as a `SYSTEM` source unit, remove only the outer brackets from `qwen_spoken_text`, and retain the bracketed source text for captions. Bracketed SFX, ambience, music, performance tags, pauses, and production directions remain non-spoken. Before TTS, compare the locked script's standalone system/UI line count with `qwen_generation_plan.json` and block on unexplained omissions.
- Streamer scripts need explicit TTS-only homograph handling for phrases such as `live content`, `streaming live`, `live stream`, and `stream content`. Prefer source/chatbot wording like `livestreaming`, `livestream`, `live broadcast`, `stream videos`, or `livestream video content`; if the approved script still contains risky wording, targeted speakability may add narrow `qwen_spoken_text` replacements without changing captions or `script_clean.md`.
- Targeted script speakability is the preferred TTS guidance stage. It must not mutate `script_clean.md`; it writes `script_speakability_report.json`, `tts_spoken_overrides.json`, and `script_speakability_problem_areas_report.json`. It should fix known problem areas only, such as explicit narrator self-reference ("the narrator wants you to understand") that sounds artificial when spoken. Broad speakability can introduce unnatural rewrites and should only run on explicit operator request.
- Do not run SFX or score planning before final narration exists.
- Do not run SFX or score planning without current local Whisper timing.
- Segment or Qwen timing is fallback metadata only; production SFX/score plans must be stamped with `timing_source: "local_whisper_word_timing"`.
- Default preflight audio target is `narrator_only`. For narrator-only runs, `run status` must skip `sfx_score_plan` and point the next audio command to `audio longform-bed --narration-only true`. Do not run audio enrichment, score generation, SFX bank resolution, or transition SFX unless the operator explicitly opts into an audio-design variant.
- Planning LLM routes are Codex or local Qwen only. Do not use ModelsLab LLM endpoints for semantic, audio, visual reference, visual prompt, or prompt review planning. ModelsLab is a media generation provider in this workflow, not a planning backend.
- Nested Codex planning calls must go through `scripts/lib/codex-cli-runner.mjs`; never spawn bare `codex exec` from a planner. Goldflow pins these calls to `gpt-5.6-sol` with medium reasoning by default, explicitly passes the model instead of inheriting `~/.codex/config.toml`, selects a GPT-5.6-compatible Codex CLI (`0.144.0+`), and writes a `.meta.json` sidecar containing model, effort, CLI path/version, and prompt hash. Cached Codex output is reusable only when that metadata matches the requested model, effort, and prompt hash. Run `goldflow run codex-doctor` before a production planner batch when the local CLI or app has changed; use `--probe true` only when a live model call is worth the spend. Explicit overrides are `ANIFACTORY_CODEX_MODEL`, `ANIFACTORY_CODEX_REASONING_EFFORT`, and `ANIFACTORY_CODEX_CLI_PATH`.
- The pinned nested planning runner is separate from Codex Imagen. It governs text planning/review subprocesses only; built-in Codex image-generation workers still follow the isolated raster staging and import contract below.
- Image provider routing is explicit and locked at preflight. If the operator says "run it through the pipeline normally" or does not name an image provider, set `--image-provider modelslab` in `goldflow run preflight` and use the goldflow default image lane: ModelsLab Flux Klein for references and scene image cuts, with Codex/local-Qwen used only for LLM planning/review tasks. `imagegen start` must refuse a provider that differs from `run_identity.json` unless the operator explicitly approves `--confirm-image-provider true`. If the operator says "Codex Imagen", "Codex imagegen", "use Codex for images", or similar, lock that in preflight and do not run the ModelsLab route for scene cuts. If the operator explicitly requests the speed-test hybrid lane, lock `--image-provider hybrid_codex_refs_multichar`: imagegen routes all references through Codex imagegen, routes risky multi-character cuts through Codex imagegen, and routes lower-risk simple scene cuts through ModelsLab. If the operator requests Codex for the retention opening only and wants references through Codex too, lock `--image-provider hybrid_codex_opening_modelslab_rest` and persist the window with `--codex-opening-sec <seconds>` in `run_identity.json`: all references use Codex imagegen, scene cuts before that timestamp use Codex, and all later scene cuts use ModelsLab. If the operator requests Codex references plus a Codex opening window plus Codex risky shots, lock `--image-provider hybrid_codex_refs_opening_risky_modelslab_rest --codex-opening-sec <seconds>`: references are generated by Codex imagegen, scene cuts before the locked timestamp and risky multi-character or explicitly Codex-routed cuts are staged/imported from built-in Codex imagegen workers, and later simple cuts are generated by ModelsLab with `--provider-filter modelslab`. If the operator requests ModelsLab references plus Codex opening scene cuts, lock `--image-provider hybrid_modelslab_refs_codex_opening_modelslab_rest --codex-opening-sec <seconds>`: references are generated by ModelsLab, scene cuts before the locked timestamp are staged/imported from built-in Codex imagegen workers, and later scene cuts are generated by ModelsLab with `--provider-filter modelslab`. Hybrid lanes are experimental provider lanes, not the normal default.
- For narrator-only runs, `sfx_score_plan` is the skipped stage. `audio longform-bed --narration-only true` is still required and is the final audio bed consumed by render; do not skip it.
- Narrator-only longform audio should write/use the final M4A and avoid retaining giant intermediate WAVs unless `--keep-wav true` is explicitly requested for diagnostics. Use `goldflow run cleanup --episode-dir <episode-dir>` to audit safe reclaimable intermediates and `--apply true` to prune them; do not delete TTS segment WAVs or rerun archives as part of normal cleanup unless explicitly requested.
- After final QA, `goldflow run cleanup` may remove redundant silent/normalized/pre-final MP4s, fallback subtitle/engagement frame directories and overlay movies, and final-mux temporary audio. It must preserve `assets/render-work/motion-clips` and their cache sidecars so a later scoped rerender remains fast.
- Fish narration artifacts are legacy/diagnostic only. Qwen production voice planning must write `qwen_generation_plan.json`, `audio_performance_plan.json`, `voice_direction_strategy_<episode>.json`, and `voice_reference_completeness_report.json`; it must not emit `narration_fish_*` or `fish_reference_requirements_report.json` unless `--emit-legacy-fish-artifacts true` is explicitly requested for a bakeoff or migration audit. Use `goldflow run cleanup` to prune old Fish files from Qwen runs.
- Codex Imagen means the built-in Codex image generation tool path, executed by Codex subagents/workers for concurrency. The production route is: assign scoped cut IDs to workers, have each worker use built-in Codex image generation to create a real raster, save exactly one PNG named for the assigned image ID inside an isolated staging folder, report the SHA-256, then import with `goldflow imagegen import-staged-codex`. Do not use nested `codex exec --enable image_generation` as the production Codex Imagen route; if it returns JSON/text such as `{"status":"success"}` without a raster, that is a failed provider probe. Do not copy the newest/global `CODEX_HOME/generated_images` file, do not mix stale staged images, and do not fall back to ModelsLab unless the operator explicitly approves that fallback in the current run.
- ModelsLab image generation is resumable by design. After a partial failure, rerun the same `imagegen start` command or retry only the failed cut ids at lower concurrency. Never use `--force` unless intentionally replacing a known-bad image, never rerun upstream visual planning/review/harden between retries for the same cuts, and never restart the pipeline from an earlier stage to recover image failures.
- Every imagegen invocation writes an immutable report under `reports/imagegen-batches/`. The materialized `imagegen_report_<episode>.json` may summarize the current episode, but cumulative spend, wall time, retries, and prior batch outcomes come from immutable history. A successful scoped batch cannot mark the episode passed while `cut_execution_ledger.json` remains partial.
- Premium image routes must pass representative text-only, one-ref, two-ref, and four-ref probes before the full queue. Three consecutive infrastructure failures open the provider circuit, preserve completed images, and mark unclaimed cuts for scoped retry. Never recover a missing image by copying a nearby donor or perturbing its hash. Deliberate editorial reuse is allowed only when metadata records `editorial_reuse_approved` and its source image id.
- `cut_execution_ledger.json` is the per-cut invalidation source: beat hash, authored/submitted prompt hashes, attached ref ids and hashes, provider/model, image hash, image-QA status, motion-profile hash, and motion-clip hash. One repaired cut invalidates one image and one motion clip; it does not invalidate the episode plan.
- ModelsLab/Flux Klein scene image generation supports up to four reference images. There is no minimum reference count and no forced lower default: attach the refs that genuinely matter for the cut, ordered by importance. Prefer visible named character/state refs first when identity or wardrobe matters, then location/environment, then critical prop/UI/action refs. Describe lower-priority characters, props, UI, and background details in text when slots are tight.
- ModelsLab GPT Image 2 can be used as a spend-forward premium image lane by setting `ANIFACTORY_IMAGE_MODEL=gpt-image-2-t2i` and, when desired, `ANIFACTORY_REFERENCE_MODEL=gpt-image-2-i2i` or the same GPT Image 2 family model. The ModelsLab helper must call v7 images endpoints, use a `size` value such as `2048x1152`, auto-route referenced cuts to `gpt-image-2-i2i`, and still verify actual landscape output before render. Use this lane only when the operator explicitly requests GPT Image 2 or a ModelsLab-credit burn test.

- SFX-only is an approved production route for narration-driven humiliation/system stories. Run audio enrichment with `--sfx-only true`, generate no score beds, and mix with `--skip-score true` so the final audio is narrator plus selected SFX only.
- SFX generation prompts must describe the concrete sound source, material, action, space, duration feel, and intensity. Do not send story summaries as SFX prompts. A style ref should show rendering style only: environments, lighting, color, linework, UI glow, fabric/material swatches, impact-light examples, or background panels. Do not use a style ref with a specific face, body, pose, or character, because it will collapse later character identities into that face.
- Run visual prompt authoring in small parent-scene-aware chunks for both Codex and local Qwen. Codex visual authoring defaults to four-cut chunks with up to six chunk calls in parallel. Scale throughput by concurrency, not by large single prompts. Optional blocker-only or diagnostic review also uses controlled chunk concurrency.
- Visual prompt review is the only LLM prompt-fix stage before imagegen, but it runs only for harden-named blockers by default. It may revise prompt wording and reference choices for those cuts, while preserving scene IDs, image IDs, timing, and source hashes.
- Visual prompt harden runs immediately after authoring, or after a blocker-only reviewed artifact. Production mode is detect-only for prose plus manifest/reference/format normalization. It writes `section_image_prompts_hardened.json`, `visual_prompt_hardening_<episode>.json`, and `visual_prompt_hardening_sample_<episode>.md`. It validates approved ref IDs and paths, strips unknown or forbidden refs, reports/block manifest-reference contradictions, enforces the four-ref limit by blocking over-limit plans, normalizes manifest/reference structure plus format-only prompt syntax, and blocks unresolved manifest/ref risks. It must not creatively infer missing locations, add characters, rewrite action, choose shot jobs, insert staging clauses, reprioritize refs, or repair narrative intent.
- `visual harden` may consume `visual_manual_blocker_triage_<episode>.json` as an explicit ledger input. Approved manual triage can mark exact blocker findings as `manual_disregard` only when the active agent has recorded cut-level evidence that the detector's suggested fix would be worse than proceeding. Harden must include the triage path, applied count, and dispositions in its report/source hashes so downstream stages know the pass depended on a reviewed manual decision.
- Localized prompt errors belong in `visual review --auto-resolve`, not in `visual harden`. The intended repair stack is: LLM-authored span repair for localized blockers, diff-guard and revalidation, harden-feedback auto-resolve for manifest/ref/location blockers found only by harden, then escalation to scene/cut re-plan, then manual agent review if capped harden-feedback blockers may be detector false positives, then dead-letter for unresolved non-harden blockers. Deterministic code may triage by structured blocker code and validate diffs; it must not patch prompt prose with regex or memorized strings.
- Prompt hardening contract (do not violate):
- Deterministic stages NEVER author or substitute depicted prompt content (locations, wardrobe, scene, character, beat prose). No story-specific names or `.replace()` on episode text anywhere.
- visual harden is DETECT-ONLY for prose: it normalizes structured manifest/reference fields and format only, and writes prompt prose byte-identical to the reviewed prose. It must not rewrite what the image depicts.
- All prose fixes happen in ONE place: visual review's `--auto-resolve` loop, where deterministic findings route to the LLM. Detectors live in `scripts/lib/prompt-prose-findings.mjs` and review; harden does not fix prose, it only asserts and, when capped harden feedback remains ambiguous, calls for manual agent review instead of pretending a detector judgment is creative truth.
- Never add a deterministic prompt-prose rewriter. If a new prose failure mode appears, add a precise DETECTOR (warning by default, blocker only when it truly breaks imagegen) to the review loop, never a `.replace()` in harden.
- `section_image_prompts_hardened.json` means "validated + manifest/format-normalized, prose intact", not "rewritten."
- Span-repair triage contract: blocker codes such as `out_of_scope_ref`, `out_of_scope_location`, `duplicate_character`, `lookalike`, and `forbidden_ref_attached` are span candidates when they carry a target field such as `location_clause`, `people_clause`, or a manifest ref field. Generated-image spelling/text accuracy is not a blocker. Structural blockers such as `wrong_shot_job`, `repeated_tableau`, `wrong_beat`, and `must_show_mismatch` route to cut/scene re-plan. Unknown blocker codes default to cut scope, never span.
- Span repair diff-guard contract: accepted span patches must preserve `image_id`, `scene_id`, `visual_beat_id`, timing, source hashes, unrelated prompt sentences, and unrelated provider prompt variants. Ref-id changes are allowed only when the blocker permits removing an offending id or adding the scoped replacement. No new out-of-scope locations, proper nouns, or visible-character changes may be introduced unless the blocker is the duplicate/lookalike case. Guard failure discards the patch and escalates to cut re-plan.
- Harden decommission migration order is load-bearing. Part F ships before Part G. For Part F: instrument branch hits first, classify real existing prompt plans second, delete only provably dead episode-prose branches third, promote live generic rules into `scripts/lib/harden-generic-rules.mjs` fourth, then add a lint wired into `npm run check` so episode-specific `.replace()` prose cannot re-accrete. For Part G: add span repair to the existing auto-resolve loop only after harden has stopped being the patch graveyard.
- Do not regenerate production episodes while implementing harden decommission or span repair. Measurement/classification runs may harden existing prompt plans in scratch outputs only. Commit Part F and Part G separately, with classification artifacts auditable when deletion happens.
- Recurring protagonists and other reusable cast belong in `canonical_entities.json` and should be referenced by stable ids. Joey is a canonical recurring MC, but literal episode sentences containing Joey are still episode contamination and must not be embedded in code. Generic rules should consume canonical entity ids, not character-name prose.
- Visual reference planning follows the same authoring boundary. Broad semantic and beat observations go into `reference_evidence_ledger.json`; explicit physical-location scope goes into `location_contract_ledger.json`. Chunk calls see bounded evidence and only propose candidates. One global merge/director call sees all chunk proposals plus both ledgers and is the sole creative selector. `reference_inventory_ledger.json` is written afterward from those final selections. Beat-level `ref_needs` are advisory evidence, not locks. Do not reintroduce keyword-based location seeding, deterministic generation budgets, or post-LLM target restoration.
- Chunked visual reference planning must not let each chunk behave like a separate art director. Chunks run concurrently (default six workers, configurable with `--visual-ref-chunk-concurrency`) and return bounded candidates with stable evidence ids. The global director may merge, reject, or rename them. An empty local chunk is valid; an empty global selection is a planner failure, not permission for deterministic fallback.
- Mixed-location scenes need beat-level textual location contracts. Parent-scene continuity is not enough when a scene spans more than one environment; each cut must lock `shot_manifest.location_contract_id` from its own beat excerpt. It may additionally attach a matching in-scope location image ref when one exists. Do not hardcode beat-keyword-to-location mappings or use an image-ref id as a substitute for text-only location truth.
- Later-timeline or status-change beats outrank stale parent-scene locations during LLM authoring/review. When a beat places the protagonist in a different environment than the parent scene began in (a return, relocation, or reversal), select that beat's current textual location contract and attach a matching image ref only when available. Decide this from the beat's own location excerpt and scoped contract set, not from enumerated plot nouns.
- Body-state proof beats in transformation stories should not repeat the next workplace/action shot. If the beat is about weight, shirt tension, grooming, posture, or visible physique, the LLM should stage a close body-detail or mirror/elevator/lobby composition that proves the current state before moving into the next job/gym/action location.
- Do not let location refs leak across cuts. If a cut attaches a ref from a different physical environment than the current beat excerpt describes, treat it as a prompt author/review failure before imagegen. Generic modern locations are opt-in from the current beat; do not use a parent scene's only location candidate as fallback when the beat itself is a montage, phone call, document insert, or abstract status beat.
- Do not import named locations, worlds, eras, or institutions from a convenient reference into an unrelated beat. If a ref is named for a later setting and the current beat/scene location does not place the action there, the LLM reviewer should remove that ref and stage the generic current location from the beat text.
- Non-character refs with `scene_ids` are scoped contracts. Location, prop, UI, and action/effect refs may be used only in their listed scenes or inclusive scene range; out-of-scope refs indicate location/world leakage and must be removed during LLM review rather than replaced by deterministic creative guessing.
- Scene scope lists with three or more scene IDs are explicit lists, not ranges. Only a two-ID `scene_ids` list is treated as an inclusive range. This prevents an apartment/location ref that lists many intended scenes from becoming falsely in scope across unrelated mixed-location montage scenes.
- `reference_usage` is review bookkeeping, not an attachment list. Code gates should validate attached refs from `reference_requirements` and manifest fields; do not block solely because `reference_usage` mentions an out-of-scope ref as removed, skipped, or considered.
- When no approved in-scope location ref exists for a beat, the LLM should stage a concrete generic location from the beat text and report a warning. Missing visible character refs, broken required paths, and unknown ref IDs still block.
- Visual prompt hardening follows the same rule: physical-location prompts without an attached location ref block only when an approved in-scope location ref exists. If no in-scope location ref exists, hardening should warn and preserve the LLM-authored concrete generic location rather than inventing a location.
- Montage parent scenes must be split into one present-tense physical location per cut before imagegen. If a scene summarizes several environments or times, each cut should keep only the beat-relevant location and props; do not allow the whole montage to appear as panel grids or multi-time collages.
- Do not attach a character ref merely because the character is mentioned in a phone message, voicemail, document, profile, memory, or possessive phrase. Attach character refs only for physically staged characters in the current cut; otherwise stage the phone, document, or abstract memory effect.
- Do not attach confrontation characters during approach/setup beats before they are physically revealed. If the current beat is the protagonist entering a building, riding an elevator, carrying takeout, or hoping someone will remember him, keep the protagonist and approach location only; add the spouse/boss/rival refs only when the current beat shows them in the room.
- Phone, voicemail, cold-email, and remote-sales-call cuts may use intentional manga-style split panels when that improves readability. Keep panel roles controlled: the largest physical panel shows the local protagonist in the real current location, while small secondary panels show device close-ups, contact avatar icons, envelope icons, phone UI shapes, opportunity badges, or abstract call-flow cards. Do not stage the remote caller as a physical person, full portrait, separate office scene, or second character unless the script beat physically places that person in the room. The failure mode is uncontrolled panels that invent people, duplicate the protagonist, add speech bubbles, or turn a remote mention into a physical scene.
- Transformation stories need enough protagonist state refs to show trajectory. Do not reuse a rain-night/discarded outfit ref for later self-improvement beats. Add intermediate refs such as early discipline, emerging founder, and final owner when wardrobe/body state changes are part of the viewer promise.
- Every visual cut must have a concrete shot job from the beat excerpt: location establishment, object insert, interaction, physical action, emotional reaction, consequence, or transition. Repeated "character at desk with UI" prompts across adjacent beats are a prompt-plan failure even if the scene context is technically correct.
- Physical-action prompts must put the decisive relationship first: who acts, what body/object is affected, where each subject is positioned, and what visible result proves the beat. Character anchors, environment, atmosphere, and style follow that action sentence. For difficult contact geometry such as lifting, pinning, stabbing, shielding, catching, or severing, state the exact contact plane and visible consequence before secondary characters or scenery. If image QA shows a neighboring setup instead of the beat, repair and regenerate only that cut; never rerun the full visual plan for an isolated action miss.
- For adult attraction/status stories, female character refs and visible-scene prompts should carry strong non-explicit visual appeal: striking faces, glossy hair, polished makeup, flattering story-appropriate outfits, conventionally attractive adult proportions, full bust, curvy hourglass silhouette, graceful waist-to-hip shape, and glamorous/high-status anime manhwa energy when the scene calls for it. Keep the language adult-coded and avoid childlike features, nudity, lingerie, explicit posing, or photorealism.
- Character refs should explicitly state that the style reference is for rendering only and that face, body, silhouette, hair, wardrobe, and expression come from the character prompt. For female adult refs, emphasize beautiful facial features, polished styling, confident silhouette, full-bust curvy hourglass adult figure, and story-appropriate attractiveness in non-explicit fashion terms.
- For real public creator likenesses, reference planning must prefer `base_identity_ref_id` plus `identity_usage: "face_only"` from the source-face anchor. The visible state ref should still be a single-person, plain-background, landscape anime/manhwa card. If no approved face source exists, block or flag for operator review instead of inventing a generic lookalike.
- Image generation uses ModelsLab Flux Klein by default when `run_identity.json` locks `image_provider: "modelslab"`. References are generated first and stored under `assets/images/references`; image cuts are stored under `assets/images`. Scene imagegen defaults to `section_image_prompts_hardened.json` and requires a hardened prompt plan by default; use `--allow-unhardened-prompts true` only for diagnostics. When ModelsLab returns a queue/rate-limit error, resume with lower `--concurrency` and keep `--force` unset so fresh refs and completed cuts are reused. This default applies only when the operator has not explicitly requested Codex/OpenAI image generation.
- For an explicit Codex Imagen run, preserve the same LLM-authored visual plan, ref manifest, timing, audio, render, and QA flow, but route image creation through built-in Codex image generation workers and import the real rasters with `goldflow imagegen import-staged-codex`. Do not use nested `codex exec --enable image_generation`, do not copy the newest/global Codex cache image, and do not substitute ModelsLab for Codex-scoped refs or cuts unless the operator approves that fallback in the current run. Import reports with `missing_image_count > 0` are proof batches only and must not be used for a full render.
- Codex/OpenAI manual or subagent scene-cut generation must be one cut per isolated staging folder. Do not actively generate into shared wave folders, and do not copy the newest/global Codex cache image. Each worker must stage exactly one file named for its assigned image ID, report its SHA-256, and the parent/import gate must reject any source hash that already belongs to a different accepted image ID. Repair attempts use a fresh isolated folder such as `wave_repair_<cut>_attempt_<n>`, not the original wave folder.
- Before render, run an accepted-image duplicate-hash audit against `imagegen_report_<episode>.json`. A render is blocked if any two distinct image IDs in the accepted report point to byte-identical rasters, even when the report has `missing_image_count: 0`; regenerate the wrong/stale cut and re-audit before rendering.
- Keep reference-image provider and final scene-image provider consistent only when the locked lane is a single-provider run. Hybrid lanes are explicit exceptions: `hybrid_codex_refs_multichar`, `hybrid_codex_opening_modelslab_rest`, and `hybrid_codex_refs_opening_risky_modelslab_rest` deliberately generate all references with Codex, then route only the approved scene-cut subset to ModelsLab. In those hybrid lanes, the hardened prompt plan and provider-aware prompt fields are the contract; do not regenerate Codex-approved refs with ModelsLab just because later scene cuts use ModelsLab.
- Scene imagegen should consume `section_image_prompts_hardened.json`. For bad cuts, regenerate only the affected `--cut-ids` with cache reuse; use `--force-images true` when replacing scene cuts while keeping approved refs fresh, and use `--force-references true` only when intentionally rebuilding refs. After targeted regeneration, run one full imagegen pass to refresh the complete image report and confirm all expected cuts exist.
- `run status` must keep image generation resumable and scoped. If derived refs are pending, it should point to `imagegen start --seed-derived-refs true --cut-ids <seed_cut_ids>`; when seed cuts exist, it should point to `imagegen promote-derived-refs`; when a report has failed or missing cuts, it should point to `imagegen start --cut-ids <failed_or_missing_ids>`. A one-cut harden/imagegen failure must never cause a full `visual plan` rerun unless the prompt plan itself is missing, stale, corrupt, or explicitly approved for full replan by the operator.
- Optional engagement bait belongs in a sparse render-layer overlay plan, not in source prose, TTS text, scene image prompts, subtitles, or audio/SFX planning. When the operator asks for comment/like/subscribe prompts, run `goldflow visual engagement` after prompt hardening and before render. Let Codex/local-Qwen choose story-timed engagement moments from the hardened visual beats, but render must apply the exact bubbles deterministically from `engagement_overlay_plan_<episode>.json` so spelling, timing, animation, and subtitle-safe placement are controlled. Keep it rare and hand placed: one early comment question after the cold open, a few choice/prediction baits on real dilemmas or reveals, and at most one or two direct like/subscribe CTAs near high-retention or end/cliffhanger moments.
- Render should use the hardened scene prompt artifact, the continuous mixed audio, final-script subtitles timed to Whisper, optional engagement overlays when deliberately planned, and final MP4s must be upload-safe `yuv420p`; verify with `ffprobe` after render.
- Image output QA is mandatory for new preflights. `goldflow imagegen qa` creates ordered five-minute sheets, a risk sheet, and `image_output_review_decisions_<episode>.json` containing the exact hash of every opening, physical-action, dense-cast, four-reference, and routed hero/risk cut. A Codex agent or operator must record `accepted`, `rejected`, or `not_inspected` for every listed cut; v2 runs do not permit bulk `--approve true`. Rejected cuts route to one scoped `imagegen start --cut-ids ... --force-images true` retry and invalidate only those cuts' motion caches. Low-risk cuts may pass structural QA automatically. Generated-image spelling is explicitly out of scope and is not a blocker.
- ModelsLab scene generation starts at its proven concurrency of 15. Queue, rate-limit, gateway, or timeout evidence automatically reduces admission concurrency for the active batch and successful requests gradually restore it; the provider circuit breaker still prevents a failing route from consuming the whole queue.
- Thumbnail packaging is separate from production scene imagegen. Use Codex/OpenAI image generation by default for final thumbnail base art because CTR packaging benefits from purpose-built composition, not a random scene still. Use episode refs/stills as visual context when useful, but compose for click readability: one dominant emotional subject, one clear reversal or humiliation target, exaggerated facial readability, high contrast, no baked model text. Add exact text, arrows, labels, and badges locally after generation so spelling and style are controlled.
- Proven channel packaging hooks outrank generic recap phrasing. The current strongest recipes are humiliation and simp framing: "wanted to humiliate me", "world's biggest simp", "my system awakened", "regressed", "left me for boss/rich man", and concrete sacrifice/debt proof. Titles should pair a simple humiliation/simp premise with a clear reversal. Thumbnail overlays should use short bubble-letter labels, yellow bubble arrows with white glow and black outline, and mobile-readable words such as `SIMP`, `LOSER`, `SYSTEM ACTIVATED`, `HUMILIATED`, `REGRESSED`, or a single concrete debt/amount when story-critical.

## Current Production Models And Methods

- Source: polished narration prose from the operator/chatbot, with exact-hash script approval.
- Speakability: targeted-only by default; broad speakability is opt-in.
- Voice: narrator-only default through ModelsLab Qwen TTS using the locked narrator voice from run identity; fallback/default is the original `joel_owned_narrator_clone`.
- Timing: local Whisper word timing is production truth for subtitles, SFX, score, visual beats, and render.
- Audio planning: Codex or local Qwen only. If automated planner calls are unavailable, a Codex-agent manual plan may be written with explicit provenance, current source hashes, and Whisper timing.
- SFX generation: ModelsLab `/api/v7/voice/sound-generation` assets are allowed after a Codex/local-Qwen/agent-authored plan. SFX events must use locked asset paths before mix.
- Ambience generation: also ModelsLab sound-generation, represented as loopable SFX events with `recurrence_class: "ambience"`, `loop: true`, short generated `asset_duration_sec`, and longer timeline `duration_sec`.
- Score generation: prefer local ACE-Step 1.5 for moment score drops and any intentionally chosen chapter beds; current default local model pair is `acestep-v15-turbo` plus `acestep-5Hz-lm-1.7B`. Score drops are short Whisper-timed ACE-Step riser/hit accents for 20-35 drama, hype, reveal, and payoff beats, mixed by ducking the base score bed if one exists.
- SFX-only mix: use `--sfx-only true` during audio enrichment and `--skip-score true` during longform-bed when the story is stronger with clean narration and punctuation SFX instead of score.
- Default image generation: ModelsLab Flux Klein, with normal production prompts, clean conditioning refs, explicit reference slot mapping, provider probes on premium routes, and concurrency 15. Explicit Codex/OpenAI image requests stay on the Codex/OpenAI lane and require real-raster verification before any batch spend. Explicit hybrid speed-test requests use `--image-provider hybrid_codex_refs_multichar`, where references and risky multi-character cuts use Codex imagegen while simpler cuts use ModelsLab. Explicit retention-window hybrid requests use `--image-provider hybrid_codex_opening_modelslab_rest --codex-opening-sec <seconds>`, where all references use Codex imagegen, scene cuts before the locked timestamp use Codex, and the rest of the video uses ModelsLab. Explicit Codex references plus opening plus risky-shot requests use `--image-provider hybrid_codex_refs_opening_risky_modelslab_rest --codex-opening-sec <seconds>`, where all references, opening-window cuts, and risky multi-character or explicitly Codex-routed cuts use Codex imagegen while later simple cuts use ModelsLab.
- Render: one continuous mixed audio track, phrase-aware final-script subtitle text timed by Whisper, yellow subtitle text with a small black outline and no background box. Run `goldflow visual motion-plan` after image QA. Its hash-bound plan derives focal anchors from LLM-authored `shot_manifest` staging or explicit image-QA overrides; missing intent becomes a smooth static hold. Production motion must never use hash-random direction fallback. `smooth_fast_ken_burns` remains the 60 fps, no-heavy-oversampling premium renderer, with hash-cached clips keyed to the directed plan. The older oversampled `fill_ken_burns` path is legacy/diagnostic only.
- Render timing is absolute. Preserve the `start_sec` timeline from `section_image_prompts_hardened.json` / `visual_beat_plan.json`; do not globally scale image durations to fill the audio. If the visual plan has a gap before the next cut, hold the current image until the next cut's real start time or add new visual beats/images in a planning pass. Never fix timing drift by stretching all cuts. Each generated motion clip must be duration-probed immediately; abort on malformed clips instead of continuing a long render.
- Render applies every selected FFmpeg `xfade` boundary across the full timeline using bounded contiguous transition segments; every unselected boundary remains a hard cut. It writes per-frame motion traces and blocks direction reversals, discontinuities, malformed duration, stale image hashes, or planned/applied transition-count mismatches. Do not ask the audio SFX planner to author generic edit whooshes/pops as narrative SFX. The visual transition planner selects transition boundaries/families, and narrator-only runs keep transition SFX disabled.
- Opening retention QA must verify three layers together: visual cuts, transition SFX, and score drops. In the first 30 seconds, the render should have a visible cut every 2-4 seconds when narration supports it. From 30-180 seconds, cuts should still arrive often enough to feel hand edited, usually every 4-7 seconds when the story is actively escalating. Transition SFX should start within roughly one frame of selected visual cuts, and score drops should land on hook/ramp turns that need sustained emotion or threat. If the audio is active but the visuals barely change, regenerate/replan first-3-minutes visual beats before the full render.
- Render QA is mandatory before treating an MP4 as publishable: run `ffprobe` for duration, fps, codec, pixel format, audio codec, and size; audit rendered cut times against transcript/visual-beat `start_sec`; extract final QA frames/contact sheets; and spot-check prompt/image/ref consistency. A final MP4 with the right total duration is not enough if individual images drift from the story.
- Complete final QA through `goldflow final qa --approve true --note <evidence>`. `run status` accepts only a passed current `final_qa_<episode>.json` whose render-report hash, final-video hash, and render source hashes still match; a QA-shaped filename is not completion evidence.

## Next Run Policy

- Treat the operator/chatbot-provided polished narration story as the candidate source of truth.
- Do not run broad enhancement by default. Use targeted speakability and readiness checks after approval, and only apply TTS-only overrides or explicitly approved script edits.
- Use narrator-only voice unless the operator explicitly requests character casting.
- Use local Whisper word timing before SFX, score, visual beats, engagement overlays, subtitles, or render.
- Use local ACE-Step for production score beds. ModelsLab music generation is a fallback only when explicitly requested.
- Use the optional score-drop layer when the run needs retention accents. Normal runs can use twenty to thirty-five Whisper-timed local ACE-Step riser/hit accents; loud retention/final-review runs should use chunked score planning around 60-80 held drops with 8-18 second durations when the operator asks for more score punctuation.
- For humiliation/system-finance stories, consider an SFX-only audio pass before committing to score. Clean narration plus selective SFX may retain better than continuous beds.
- For imagegen, prioritize reference quality and spot checks over raw throughput. Start concurrency around 6-12 on the next full run, then raise only after references and prompt quality look stable.
- Babysit the next run through staged visual gates before trusting full automation: manually review references, review the sanitized prompt sample, generate and contact-sheet the first small image batches, then keep periodic contact-sheet spot checks through high-risk sections. Do not treat imagegen as set-and-forget until several consecutive batches show no duplicate protagonist, subject fusion, destructive UI, wrong refs, or major location/style drift.
- If generation times out, count missing references/cuts and resume only those IDs with cache reuse. Do not force-regenerate completed good images.
- Keep narration competitively loud from the longform mix using `--narration-volume-db 2` as a safe baseline, then verify clipping and intelligibility against score beds and drops. When the operator asks for the loud retention/final review version, use the narrator-forward recipe (`--narration-volume-db 3`, score-drop boost around `3`, target `-13 LUFS`, true peak `-1 dBTP`), dense opening narrative SFX, ambience where it fits, and moment-directed score drops rather than an always-on score bed. Transition SFX are not part of audio planning by default; the visual transition plan and render/edit layer place them on selected cuts.
